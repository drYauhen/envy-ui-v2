# generated/figma/

Purpose: Figma platform artifacts derived from tokens and platform structures.
Source/workflow: Style Dictionary platforms `figma`, `figmaScoped`, `pluginVariables`; structures currently curated manually.
Consumers/destinations: Figma plugin workflows and Figma Variables import.
Artifact types/roles: Adapter payloads, variables payloads, and structures for Figma.

**For detailed workflow documentation, see:** [`docs/workflows/figma-workflow.md`](../../docs/workflows/figma-workflow.md)

Producers
- `style-dictionary/config.mjs` platform `figma` (format `figma/adapter`)
- `style-dictionary/config.mjs` platform `figmaScoped` (format `figma/variables-scoped` - general, all contexts)
- `style-dictionary/config.mjs` platform `figmaApp` (format `figma/variables-scoped` - app context only)
- `style-dictionary/config.mjs` platform `figmaWebsite` (format `figma/variables-scoped` - website context only)
- `style-dictionary/config.mjs` platform `figmaReport` (format `figma/variables-scoped` - report context only)
- `style-dictionary/config.mjs` platform `pluginVariables` (format `figma/variables-full`)
- Figma plugin (snapshots, migrations)

Expected artifacts
- `adapter/variables.adapter.json` (adapter payload - legacy, colors only)
- `tokens/variables.tokens.full.json` (full variables payload - general)
- `tokens/variables.tokens.scoped.json` (scoped variables payload - general, all contexts)
- `app/variables.tokens.scoped.json` (app context only - app-default, app-accessibility modes)
- `website/variables.tokens.scoped.json` (website context only - website-default, website-dark modes)
- `report/variables.tokens.scoped.json` (report context only - report-print, report-screen modes)
- `structures/structures.eui.button.json` (structures payload for UI generation - frozen)
- `snapshots/` (Figma Variables snapshots created via plugin for migration tracking)
- `migrations/` (migration files for automated binding updates - generated by `scripts/figma/generate-migration.mjs`)

**Context-specific files** (`app/`, `website/`, `report/`) are preferred for import to avoid context mismatch errors.

Regeneration
- `npm run tokens:build` (builds all token outputs including general Figma files)
- `npm run tokens:build:figma` (builds all three context-specific files: app, website, report)
- Or individually:
  - `npm run tokens:build:figma:app` (app context only)
  - `npm run tokens:build:figma:website` (website context only)
  - `npm run tokens:build:figma:report` (report context only)
