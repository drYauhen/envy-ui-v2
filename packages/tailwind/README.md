# Tailwind Implementation Layer

This package provides a Tailwind CSS-based implementation of Envy UI components.

## Architecture

This layer is a **parallel implementation** to other layers (HTML+CSS, TSX, Web Components), all projecting from the same token system. Components use Tailwind utility classes generated from design tokens.

## Token Integration

Design tokens are automatically mapped to Tailwind theme values:

- **Colors**: Foundation color tokens → Tailwind color palette
- **Spacing**: Foundation spacing tokens → Tailwind spacing scale
- **Typography**: Foundation typography tokens → Tailwind font size/weight/line-height
- **Shape**: Foundation shape tokens → Tailwind border radius

The mapping is generated by `scripts/generate-tailwind-config.mjs` and written to `config/tailwind.config.js`.

## Usage

### Generating Tailwind Config

Run the generator to sync tokens to Tailwind config:

```bash
npm run tailwind:generate-config
```

This script:
1. Reads tokens from `tokens/foundations/`
2. Maps them to Tailwind theme structure
3. Generates `config/tailwind.config.js`

### Using Components

```tsx
import { Button } from '@envy-ui/tailwind';

<Button intent="primary" size="md">Click me</Button>
```

## Component Structure

Each component follows this structure:

```
components/
  {component}/
    {Component}.tsx      # Component implementation with Tailwind classes
    {Component}.stories.tsx  # Storybook stories (optional, usually in stories/)
```

## Token Mapping

### Colors

Foundation colors are flattened into Tailwind format:
- `brand.600` → `brand-600` (use as `bg-brand-600`)
- `accent.700` → `accent-700` (use as `bg-accent-700`)

### Spacing

Foundation spacing tokens map directly:
- `spacing.sm` → `sm` (use as `px-sm`, `py-sm`)

### Typography

Font sizes, weights, and line heights map to Tailwind defaults with token values.

## Storybook Integration

Components are documented in Storybook under the "Tailwind" section. The Tailwind CSS is automatically processed via PostCSS in Vite.

## Status

**Status:** Exploratory

This is an initial implementation. The token mapping and component structure may evolve based on learnings from usage.
